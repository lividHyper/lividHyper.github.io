Hugo搭建Blog的一些用法

April 7, 2023 · 4 min · livid | Translations:
  • En

经过多次的入坑弃坑，还是打算重新捡起个人 blog，一方面是希望将自己工作和生活上的一些思考分享给大家，另一方面也是考虑锻炼下写作的能力。

工欲善其事#

一直以来，作为一个工具党，在工具的挑选上总是存在选择困难，考虑到 blog 最主要的目的就是写下一些东西，这里还是采用了最简单的 Github Pages + Hugo 来构建，好友推荐的 PaperMod 的主题也是十分戳中我的审美。在这里也是快速对整个 blog 的搭建记录做一个简单的回顾。

整体的结构#

Hugo 的配置#

Hugo 本身使用 Go 语言实现，本身的配置学习起来有一定的上手成本，建议直接抄录一份现成的 yml 进行更新，以下是我的配置文件;

baseURL: https://lividhyper.github.io/
defaultContentLanguage: zh-cn
theme: hugo-PaperMod
summary: archives

enableEmoji: true
enableRobotsTXT: true

# 自动检测是否包含 中文日文韩文 如果文章中使用了很多中文引号的话可以开启
hasCJKLanguage: true

# 每页显示的文章数
pageinate: 10

# 浏览器显示方式
permalinks:
  post: "/:title/"

# 开启标签tag
taxonomies:
  category: categories
  tag: tags
  series: series
  entag: entags

languages:
  zh-cn:
    title: Livid的博客
    contentDir: content/zh

    menu:
      main:
        - identifier: archives
          name: 文章
          url: archives/
          weight: 1
        - identifier: tag
          name: 分类
          url: tags/
          weight: 10
        - identifier: search
          name: 搜索
          url: search
          weight: 20
        - identifier: collection
          name: 资料
          url: collection/
          weight: 20
        - identifier: about
          name: 关于&友链
          url: about
          weight: 30
  en:
    title: Livid's Blog
    contentDir: content/en
    menu:
      main:
        - identifier: archives
          name: Post
          url: archives/
          weight: 1
        - identifier: entag
          name: Tags
          url: entags/
          weight: 10
        - identifier: search
          name: Search
          url: search
          weight: 20
        - identifier: collection
          name: Collection
          url: collection
          weight: 40
        - identifier: about
          name: About
          url: about
          weight: 50
params:
  # 开启SEO
  env: production
  author: livid
  ShowReadingTime: true

  homeInfoParams:
    Title: xx 
    Content: xx
  comments: true

  socialIcons: # optional
    - name: github
      url: "https://"
    - name: twitter
      url: "img/twitter.png"
    - name: RSS
      url: "index.xml"

  # 允许代码被拷贝
  ShowCodeCopyButtons: true

  fuseOpts:
    isCaseSensitive: false
    shouldSort: true
    location: 0
    distance: 1000
    threshold: 1
    minMatchCharLength: 0
    keys: ["title", "permalink", "summary"]



outputs:
  home:
    - HTML
    - RSS
    - JSON # is necessary

searchHidden: true

通过 Github Actions 来实现自动发布#

我们通过配置 2 个仓库，一个用来存放文章，可以命名为 blog，允许设置为 Private 仓库， 一个用来存放对外的静态资源，这个需要设置为 Public 仓库；Github Action 的 secret 和配置不做过多赘述，这里是我的一份 .github/worklofws/deploy.yml的示例;

name: deploy

on:
  push:
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v2
        with:
          submodules: true
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: "latest"

      - name: Build Web
        run: hugo

      - name: Deploy Web
        uses: peaceiris/actions-gh-pages@v3
        with:
          PERSONAL_TOKEN: ${{ secrets.PERSONAL_TOKEN }}
          EXTERNAL_REPOSITORY: lividHyper/lividHyper.github.io
          PUBLISH_BRANCH: main
          PUBLISH_DIR: ./public
          commit_message: ${{ github.event.head_commit.message }}
