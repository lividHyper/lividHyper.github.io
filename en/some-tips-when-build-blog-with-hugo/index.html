Some tips when build blog with hugo

April 7, 2023 · 2 min · livid | Translations:

Translate By ChatGPT.

After several attempts and giving up, I finally decided to pick up my personal blog again. On the one hand, I hope to share some of my thoughts on work and life with everyone, and on the other hand, I also want to exercise my writing ability.

Work well with tools#

As a tool enthusiast, I always have difficulty choosing tools. Considering that the main purpose of a blog is to write something down, I still use the simplest Github Pages + Hugo to build it. The PaperMod theme recommended by my friend is also very much in line with my aesthetics. Here, I will make a brief review of the construction of the entire blog.

Overall structure#

Hugo configuration#

Hugo itself is implemented using the Go language, and its configuration has a certain learning curve. It is recommended to copy and update an existing yml file directly. Here is my configuration file:

baseURL: https://lividhyper.github.io/
defaultContentLanguage: zh-cn
theme: hugo-PaperMod
summary: archives

enableEmoji: true
enableRobotsTXT: true

# 自动检测是否包含 中文日文韩文 如果文章中使用了很多中文引号的话可以开启
hasCJKLanguage: true

# 每页显示的文章数
pageinate: 10

# 浏览器显示方式
permalinks:
  post: "/:title/"

# 开启标签tag
taxonomies:
  category: categories
  tag: tags
  series: series
  entag: entags

languages:
  zh-cn:
    title: Livid的博客
    contentDir: content/zh

    menu:

        - identifier: archives

          url: archives/

Automatically publish using Github Actions#

We can set up two repositories: one for storing articles, which can be named blog and can be set to a Private repository; the other for storing external static resources, which needs to be set to a Public repository. The secrets and configurations of Github Action are not mentioned too much here. Here is an example of my .github/workflows/deploy.yml:

name: deploy

on:
  push:
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v2
        with:
          submodules: true
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: "latest"

      - name: Build Web
        run: hugo

      - name: Deploy Web
        uses: peaceiris/actions-gh-pages@v3
        with:
          PERSONAL_TOKEN: ${{ secrets.PERSONAL_TOKEN }}
          EXTERNAL_REPOSITORY: lividHyper/lividHyper.github.io
          PUBLISH_BRANCH: main
          PUBLISH_DIR: ./public
          commit_message: ${{ github.event.head_commit.message }}

Image Hosting Configuration#

Since I use Github to host my blog, I will skip the configuration of an image hosting service and simply embed the images directly into my blog as static resources.

Comments#

Since my blog is not yet heavily trafficked, I will hold off on introducing a commenting system for now. I will consider implementing a more mature solution in the future.
